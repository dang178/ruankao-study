# Day 2：计算机系统基础知识（软件篇）理论知识详解

## 1. 操作系统基础

### 操作系统组成
- **内核(`Kernel`)**：操作系统核心，负责进程管理、内存管理、设备驱动和系统调用
- **系统库(`System Libraries`)**：提供应用程序与内核交互的接口
- **系统实用程序(`System Utilities`)**：文件管理、网络配置等基础工具
- **用户界面(`User Interface`)**：命令行界面(CLI)或图形用户界面(GUI)

### 操作系统主要功能
- **进程管理**：创建、调度、同步和通信
- **内存管理**：分配、回收和保护内存空间
- **文件系统管理**：组织、存储和检索数据
- **设备管理**：控制和协调硬件设备
- **网络管理**：提供网络通信能力
- **安全管理**：用户认证、权限控制和数据保护

### 操作系统分类详解
- **批处理操作系统**：
  - 特点：作业成批处理，无需人工干预
  - 代表系统：IBM OS/360
  - 适用场景：科学计算、大规模数据处理

- **分时操作系统**：
  - 特点：多用户同时共享CPU，时间片轮转
  - 代表系统：UNIX、Linux
  - 适用场景：多用户交互式系统

- **实时操作系统(`RTOS`)**：
  - 硬实时：必须在严格时限内完成任务（如航天控制）
  - 软实时：允许偶尔超时（如多媒体系统）
  - 代表系统：VxWorks、QNX
  - 适用场景：工业控制、嵌入式系统

- **嵌入式操作系统**：
  - 特点：资源占用小，专用性强
  - 代表系统：FreeRTOS、μC/OS-II
  - 适用场景：智能设备、物联网终端

## 2. 数据库系统基础

### 关系型数据库核心特性
- **数据模型**：基于集合论和关系代数，以二维表格形式组织数据
- **SQL语言**：结构化查询语言，提供数据定义、查询和操作功能
- **ACID特性详解**：
  - **原子性(`Atomicity`)**：事务中的所有操作要么全部完成，要么全部不完成
  - **一致性(`Consistency`)**：事务执行前后，数据库必须保持一致性状态
  - **隔离性(`Isolation`)**：并发执行的事务之间相互隔离
  - **持久性(`Durability`)**：一旦事务提交，其结果永久保存

### 分布式数据库特点
- **数据分布透明性**：用户无需知道数据物理位置
- **分布式查询处理**：查询优化器决定最佳执行路径
- **分布式事务管理**：两阶段提交协议(`2PC`)确保事务一致性
- **复制与一致性**：主从复制、多主复制等模式

### 数据库新技术
- **NewSQL**：结合关系型和NoSQL优势（如Google Spanner）
- **HTAP**：混合事务/分析处理（如TiDB）
- **云原生数据库**：自动扩缩容、按需付费（如AWS Aurora）

## 3. 文件系统基础

### 文件系统层次结构
- **用户接口层**：提供文件操作API
- **逻辑文件系统**：管理元数据和目录结构
- **基本文件系统**：管理磁盘块和I/O操作
- **I/O控制层**：设备驱动和硬件交互

### 常见文件系统类型
- **FAT32**：兼容性好，但单文件最大4GB
- **NTFS**：支持大文件、权限控制和日志功能
- **ext4**：Linux主流文件系统，支持大容量和日志
- **ZFS/Btrfs**：支持快照、压缩和校验和

### 文件系统关键技术
- **索引节点(`Inode`)**：存储文件元数据
- **日志技术**：记录操作过程，提高系统恢复能力
- **缓存机制**：提高文件访问速度
- **文件锁**：控制并发访问

## 4. 网络协议基础

### TCP/IP协议栈详解
- **应用层**：
  - HTTP/HTTPS：Web通信
  - FTP：文件传输
  - SMTP/POP3/IMAP：电子邮件
  - DNS：域名解析

- **传输层**：
  - TCP：面向连接，可靠传输
    - 三次握手建立连接
    - 四次挥手断开连接
    - 滑动窗口控制流量
  - UDP：无连接，低延迟

- **网络层**：
  - IP协议：寻址和路由
  - ICMP：网络诊断（ping命令）
  - ARP：IP地址到MAC地址映射

- **数据链路层**：
  - Ethernet：局域网标准
  - PPP：点对点协议

### 网络安全协议
- **SSL/TLS**：加密传输层通信
- **IPSec**：网络层安全协议
- **SSH**：安全远程登录
- **Kerberos**：网络身份验证协议

## 5. 中间件技术

### 中间件分类
- **消息中间件**：
  - 特点：异步通信、解耦系统
  - 代表产品：RabbitMQ、Kafka、RocketMQ
  - 应用场景：日志收集、事件驱动架构

- **事务处理中间件**：
  - 特点：支持分布式事务
  - 代表产品：Tuxedo、IBM CICS
  - 核心技术：两阶段提交(`2PC`)

- **对象请求代理(`ORB`)**：
  - 特点：支持对象间远程调用
  - 代表技术：CORBA、RMI
  - 核心协议：IIOP

- **Web服务中间件**：
  - 特点：基于标准协议的互操作
  - 代表技术：SOAP、RESTful API
  - 相关标准：WSDL、UDDI

### 中间件核心价值
- **系统集成**：连接异构系统，实现数据共享
- **服务复用**：避免重复开发，提高开发效率
- **性能优化**：负载均衡、连接池等机制提升系统性能
- **可靠性保障**：事务管理、故障转移等机制
